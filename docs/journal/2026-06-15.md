# Dev Journal — 2026-06-15

## Summary
Hardened the blog CMS and locked down the admin area after a security audit, plus
shipped per-post SEO controls and Persian/RTL authoring support.

## Shipped
- **Security: admin area gated server-side.** Added `middleware.ts` to cryptographically
  verify the `session` JWT (jose, HS256) for every `/admin/:path*` route except
  `/admin/login`. Before this, admin server pages — including `/admin/messages`, which
  renders contact-form PII — rendered for anyone. `AdminShell` is a client component and
  never provided real auth.
- **Security: JSON-LD injection sealed.** Escaped `<` (`<`) in all
  `dangerouslySetInnerHTML` JSON-LD on `app/[locale]/blog/[slug]/page.tsx` so a
  `</script>` sequence in any string value can't break out of the tag.
- **Security: contact-form spam protection.** Honeypot `company` field (hidden from
  humans, silent 200 if filled) + best-effort per-IP rate limit (5/min) in
  `app/api/contact/route.ts`.
- **Per-post SEO controls.** New nullable `Post` fields (`metaDescription`, `ogTitle`,
  `ogDescription`, `ogImage`, `jsonLd`, `headHtml`) with an "SEO & Social" panel in the
  post editor; emitted via `generateMetadata` (OG + Twitter) and custom JSON-LD/head HTML
  on the public page.
- **Raw-HTML body mode.** `</>` toggle in `RichEditor` swaps the TipTap surface for a
  monospace textarea, kept in sync with `PostForm.content`.
- **RTL/Persian authoring.** Per-post `dir` column, LTR/RTL toolbar buttons, Vazirmatn
  editing surface; published posts render in their stored direction.
- **robots:** block `/admin/` and `/api/`, keep AI crawlers allowed.

## Notes
- DB migrations applied via the Neon console (local Prisma CLI can't reach Neon);
  `npx prisma generate` run locally only.
- Docs corrected: `CLAUDE.md` previously claimed AdminShell enforced auth — it does not;
  `middleware.ts` is the real gate.
